19年6月1日，据广大墙内网友反馈，gfw进行了技术升级，升级之后外国vps服务器大面积ip被封，原有的SS/SSR翻墙技术全面陷入瘫痪，至此反制技术升级迫在眉睫，本文主要介绍目前最被看好的V2Ray。此技术原生支持 Socks、HTTP、Sha-dow-socks、VMess 等协议，可以在一个进程中可以配置不同的端口使用不同的协议进行通讯，通过不同的传入和传出协议组合，灵活转换通讯格式

官网：https://www.v2ray.com/ 有整体详细的技术介绍，前提是你得能访问，很可能是不行


那么我们首先需要购买一个具有独立IP地址的VPS云服务器，目前主流的VPS供应商均可，特别注意的是需要测试vps的IP是否能ping通，不能ping通则以下的步骤无法展开，ping延时越小越好


服务器端：Centos安装V2Ray

1. 使用ssh命令登录vps ssh root@xxx.xxx.xxx.xxx , 然后输入密码

2. 确认os是否有wget命令

用 which wget查看，如果成功显示了指令的安装位置则无需安装，如：/usr/bin/wget

若无显示则使用 sudo yum -y install wget 安装

3.下载v2ray服务器端安装脚本

wget https://install.direct/go.sh

4.确认os是否有zip unzip指令

若无则使用 sudo yum install zip unzip 安装

5.运行安装脚本

sudo bash go.sh 等待命令执行完成，当看到以下信息则证明安装成功（版本号会有不同）

V2Ray v4.19 is installed.

此时需要记录下安装信息中的PORT及UUID值

6.启动服务

首次安装完成之后，V2Ray不会自动启动，需要手动运行启动命令。而在已经运行V2Ray的vps上再次执行安装脚本，安装脚本会自动停止V2Ray 进程，升级V2Ray程序，然后自动运行V2Ray。在升级过程中，配置文件不会被修改。

## 启动

sudo systemctl start v2ray

## 停止

sudo systemctl stop v2ray

## 重启

sudo systemctl restart v2ray


7.配置信息修改

使用cat /etc/v2ray/config.json查看配置信息，配置信息中的id、端口、alterId需要与客户端的配置保持一致，修改这个文件之后需要重启服务


8.防火墙开放端口

服务启动成功后可能会被防火墙拦截，需修改防火墙的拦截端口

iptables -I INPUT -p tcp --dport xxxxx -j ACCEPT

xxxxx为之前服气启动时的PORT值

至此，服务器端的安装工作结束，接下来是客户端的安装配置


客户端：

下载 V2Ray

预编译的压缩包可以在如下几个站点找到：

Github Release: github.com/v2ray/v2ray-core

Github 分流: github.com/v2ray/dist

Homebrew: github.com/v2ray/homebrew-v2ray

Arch Linux: packages/community/x86_64/v2ray/

Snapcraft: snapcraft.io/v2ray-core

压缩包均为 zip 格式，找到对应平台的压缩包，下载解压即可使用。


运行之后找到配置选项

填入在服务器配置信息，IP，端口，UserID(服务器配置时的UUID)，点击OK


之后我们需要编辑自动翻墙的网站列表


手动填入需要翻墙的网站，之后再打开网站就可以自动翻墙了，至此配置基本完成，可以继续开开心心上网了
